# 穹州图片工具 — 安装说明

本文档面向**已发布的安装包**用户，不涉及源码与开发环境。

---

## 下载地址

安装包发布在公开仓库的 **Releases** 页面：

**https://github.com/ck7526296/qzkj-app-releases/releases**

进入后选择最新版本，按系统下载对应文件即可。

| 系统 | 常见安装包 |
|------|------------|
| macOS | `.dmg` 磁盘映像 |

## macOS 安装步骤

### 1. 下载

在 Releases 页面下载当前版本的 `.dmg` 文件（例如 `穹州图片工具_0.x.x_universal.dmg`，具体文件名以发布页为准）。

### 2. 打开安装包

双击 `.dmg`，在弹出的窗口中将 **穹州图片工具** 拖入 **应用程序（Applications）** 文件夹。

### 3. 首次打开与安全提示

若系统提示「无法打开，因为来自身份不明的开发者」或类似信息：

1. 打开 **系统设置** → **隐私与安全性**。
2. 在「安全性」相关区域找到被拦截的应用提示，选择 **仍要打开**；或  
3. 在 **访达** 中按住 **Control** 键点按应用图标，选 **打开**，在对话框中确认打开。

这是 macOS 对未在 Mac App Store 分发的应用的常规流程，并非软件异常。

### 4. （可选）命令行解除隔离属性

从网络下载的应用会被系统打上 **隔离（quarantine）** 标记，有时会出现「已损坏，无法打开」等提示。若图形界面操作仍无法打开，可在确认安装包来源可信的前提下，用 **终端** 清除该标记。

1. 打开 **终端**（启动台 → 其他 → 终端，或 **Command + 空格** 搜索「终端」）。
2. 若应用已安装在默认位置，执行：

```bash
sudo xattr -dr com.apple.quarantine "/Applications/穹州图片工具.app"
```

3. 若应用放在其它目录，把引号内路径改成该 `.app` 的实际位置（可将应用从访达拖入终端窗口以自动填入路径）。
4. 执行后无报错即表示已处理，再在访达或启动台中正常打开应用。

**若提示 `Operation not permitted` 或权限不足：**

- 打开 **系统设置** → **隐私与安全性** → **完全磁盘访问权限**，将 **终端**（或你使用的 **iTerm** 等）加入列表并勾选启用，然后**退出终端重新打开**再执行上述命令；或  
- 使用 `sudo`（会要求输入本机登录密码，输入时屏幕不显示字符属正常现象）：

```bash
sudo xattr -dr com.apple.quarantine "/Applications/穹州图片工具.app"
```

**安全说明：** 仅当您确认安装包来自官方 Releases 或可信渠道时再使用上述命令；不要随意对来源不明的 `.app` 清除隔离属性。

### 5. 启动

在 **启动台** 或 **应用程序** 文件夹中打开 **穹州图片工具**。

### 6. 系统与架构

- 推荐使用 **macOS 12** 或更新版本。
- 发布包一般为 **通用二进制（Apple Silicon + Intel）**，同一安装包可在 M 系列芯片与 Intel Mac 上使用（以各版本 Release 说明为准）。

## Windows 安装步骤（简要）

1. 在 [Releases](https://github.com/ck7526296/qzkj-app-releases/releases) 下载对应版本的 `.msi` 文件。

## 常见问题

**Q：安装后找不到应用？**  
A：在访达中打开「应用程序」文件夹，或按 **Command + 空格** 搜索「穹州」。

**Q：提示文件已损坏？**  
A：多为隔离属性或门禁策略导致。请先按上文「首次打开与安全提示」操作；仍不行时，可按「命令行解除隔离属性」一节用 `xattr` 处理（须确认下载来源可信）。

**Q：官网或其它下载渠道？**  
A：请以 GitHub Releases 或官方说明的下载链接为准，避免从不明来源下载安装包。
